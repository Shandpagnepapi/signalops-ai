# Free Missed Lead Check Checklist

Use this checklist to review how a business handles calls, texts, forms, DMs, and follow-ups before proposing implementation. Mark each line as `Yes`, `No`, or `Needs Work`.

## Lead Sources

- [ ] All lead sources are documented (forms, calls, ads, DMs, referrals, email).
- [ ] Source attribution is captured for each lead.
- [ ] After-hours lead coverage is defined.

## Response Time

- [ ] First response target is defined.
- [ ] Actual response time is tracked.
- [ ] Escalation path exists for urgent leads.

## Missed Calls

- [ ] Missed-call text back is active.
- [ ] Missed calls are logged in one place.
- [ ] Follow-up process exists for no-answer callbacks.

## Forms

- [ ] Form submissions are routed reliably.
- [ ] Required fields are practical and complete.
- [ ] Team receives immediate notifications on submission.

## Quote Requests

- [ ] Quote request workflow is documented.
- [ ] Quote requests are prioritized by urgency and value.
- [ ] Quote turnaround expectations are defined and measured.

## Detail Intake

- [ ] Detail request flow exists when key information is missing.
- [ ] Site, account, or photo guidelines are sent to prospects when useful.
- [ ] Fleet wash companies: fleet size, vehicle type, location, wash window, and site requirement intake is documented.

## CRM

- [ ] Leads are logged consistently in CRM.
- [ ] Status stages match real sales workflow.
- [ ] Pipeline is updated frequently enough for decision-making.
- [ ] If there is no CRM, calls, texts, forms, and DMs still have a simple tracking process.

## Calendar Booking

- [ ] Booking link or scheduler is connected to lead flow.
- [ ] Appointment routing rules are defined by lead type.
- [ ] No-show and reschedule follow-up exists.

## Follow-Up

- [ ] Follow-up cadence exists beyond first response.
- [ ] Follow-up messages are channel-appropriate (SMS/email/call).
- [ ] Stop conditions are defined (booked, disready, no response).

## Sales Handoff

- [ ] Owner/sales rep receives clear lead context before outreach.
- [ ] Handoff includes summary, priority, and recommended action.
- [ ] High-priority leads trigger immediate alerts.

## Reporting

- [ ] Lead volume by source is tracked weekly.
- [ ] Response time and booking rate are visible.
- [ ] Lost-lead reasons are captured.

## AI Opportunities

- [ ] Instant first response can be automated safely.
- [ ] Intake prompts can be automated by service type.
- [ ] Priority sorting criteria are clear enough to operationalize.
- [ ] Owner alert rules can be defined for hot leads.
- [ ] Follow-up sequences can be triggered automatically.
- [ ] Uncertain or risky scenarios are routed to human review.
